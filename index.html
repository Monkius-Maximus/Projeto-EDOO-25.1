SGMP

Sistema de Gerenciamento de Manutenção Predial em C++

Arquitetura do Sistema

O SGMP foi desenvolvido com foco em modularidade e organização. Ele permite que moradores de um condomínio registrem chamados de manutenção, que são atribuídos a técnicos e acompanhados até a finalização.

  • Pessoa: classe base para usuários (ID, nome, contato)
  • Morador: herda de Pessoa (inclui bloco e apartamento)
  • Técnico: herda de Pessoa (inclui especialidade)
  • OrdemDeServico: gerencia status, data, descrição, técnico e morador envolvidos

Aplicação dos Conceitos de POO

Herança: permite reutilizar código com a classe base Pessoa, herdada por Morador e Técnico.

Polimorfismo: cada tipo de usuário implementa o método exibirInformacoes(), que é chamado dinamicamente.

Encapsulamento: todos os atributos são privados e acessados via métodos públicos, mantendo a integridade dos dados.

Fluxo de Uso do Sistema

  1. Morador acessa o sistema e abre uma Ordem de Serviço (OS).
  2. O sistema cria a OS com status "ABERTA".
  3. Um Técnico com a especialidade correspondente vê a OS e se responsabiliza.
  4. Status muda para "EM ANDAMENTO".
  5. Após o reparo, técnico finaliza a OS, mudando o status para "CONCLUÍDA".
  6. O histórico completo é salvo e pode ser consultado.
